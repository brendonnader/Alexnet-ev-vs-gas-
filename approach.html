Model Architecture: AlexNet

AlexNet is a convolutional neural network (CNN) that became famous after winning the ImageNet competition in 2012. In this project, I use a pretrained AlexNet model from PyTorch and fine-tune it on my electric vs gas vehicle dataset.

Key components:

  • Convolution layers to detect edges, textures, and shapes.
  • ReLU activations for faster non-linear learning.
  • Max pooling layers to downsample feature maps.
  • Fully connected layers that map learned features to class scores.

Layer-by-Layer Summary

  • Conv1 + ReLU + MaxPool
  • Conv2 + ReLU + MaxPool
  • Conv3 + ReLU
  • Conv4 + ReLU
  • Conv5 + ReLU + MaxPool
  • Fully connected (4096 units) + ReLU + Dropout
  • Fully connected (4096 units) + ReLU + Dropout
  • Output layer: 2 units (Electric vs Gas) replacing the original 1000-class layer.

Training Process

  • Input size: 3 × 227 × 227 (RGB images).
  • Preprocessing: Resize, random horizontal flip (data augmentation), convert to tensor, and normalize using ImageNet mean and std.
  • Loss function: Cross-Entropy Loss (standard for classification).
  • Optimizer: Adam with learning rate 0.001.
  • Batch size: 32.
  • Epochs: 10 (can be increased for better performance).
  • Datasets: Training and validation sets are loaded from separate folders using PyTorch’s ImageFolder and DataLoader.

How the Data Fits the Model

Each image from the dataset is transformed into a tensor with shape [3, 227, 227]. The last fully connected layer of AlexNet is replaced with: 

model.classifier[6] = nn.Linear(in_features, 2)

This tells the network to output probabilities for the two classes: Electric and Gas.
